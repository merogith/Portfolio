this file holds the *raw material* you paste while tailoring a one-pager:
three ready-made positioning variants, a keyword bank per role-type, and the
quick tests to run before you send anything.

Same ground rule as the rest of the system: **every fact below is already on
`master-cv.html`.** Never paste a number, date, title or tool that isn't.

## 4. Pre-send checklist (run every time)

From the best-practice review — cheap tests that catch the real failure modes.

- [ ] **Plain-text paste test.** Copy the printed PDF's text, paste into a plain
      editor. If the order scrambles or sections vanish, the ATS sees the same mess.
      (The CV pages are single-column true-text, so this should stay clean.)
- [ ] **Title-line matches the ad.** Mirror the posting's job title exactly.
- [ ] **Keyword coverage.** Each must-have term from the ad appears in context.
- [ ] **Knockout questions.** Assert EU work authorization / dual citizenship; answer
      location, language and years-of-experience screens precisely and favourably.
- [ ] **6-second test.** Hand it to someone for 6 seconds — can they name the target
      role and your top result?
- [ ] **"So what?" test.** Every bullet ends in an outcome or a number.
- [ ] **File name** = `Meric_Erler_CV_<Role>.pdf` (role in the name, not `v3_final`).
- [ ] **Format.** DOCX/true-text PDF for international & remote-EU tech; clean
      single-column PDF for German applications.
- [ ] **One page**, prints clean, no orphan lines pushing onto a second sheet.

> File naming note: `assets/CV_Meric_Erler.pdf` keeps a stable name on purpose —
> the site links to it. The *per-application* one-pagers are the ones you rename
> per role before attaching.
