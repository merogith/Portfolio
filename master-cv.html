US Stock-Market Valuation & Risk Python · OLS · SQL · Tableau

150+ years of real market data — does valuation predict long-run returns, and how much risk do you sign up for?

  • Pulled Robert Shiller's price / earnings / CAPE series plus CBOE (VIX) and LBMA (gold) with a reproducible script; the cleaning call that mattered — recent months store 0.0 as a placeholder for unreported fundamentals, so I set those to missing rather than let them drag every average down.
  • Built a forward 10-year real-return series in SQLite (a LAG window function to rank the best and worst years) and regressed forward returns on starting CAPE with OLS in statsmodels.
  • Higher valuation tracks lower forward returns — statistically clear but R² = 0.11 (a real signal worth ~a tenth of the variation, not a market-timing tool); today's CAPE of 30.8 sits in the top 5% since 1881.
  • Quantified the price of admission: real growth of only ~2.4%/yr once inflation is stripped, a worst real drawdown of −81% (trough June 1932), and a stock–gold correlation ≈ 0 — a genuine diversifier.
  • Shipped as a Python notebook and an interactive Tableau dashboard on a star-schema export.
  • Python
  • statsmodels (OLS)
  • SQL window fns
  • SQLite
  • Tableau
  • matplotlib
Pharmaceutical CRM Sales Analytics Python · scikit-learn · SQL · Tableau

Do sales calls drive prescriptions, and where should a rep spend limited time across 600 physicians? A shareable version of my real Viatris CRM work — on synthetic data.

  • Ran on a synthetic 600-physician CRM dataset from seeded scripts that deliberately inject real-world mess (duplicates, missing values, mixed date / text formats), so the cleaning steps are real, not decorative.
  • Tested the call → prescription link with correlation and OLS, segmented doctors with KMeans, and built a ranked priority list with RANK() OVER (PARTITION BY region …).
  • Calls move scripts — r = 0.49, about +5.5 scripts per extra call — and prescribing is concentrated (top 20% of doctors write ~a third of scripts), so which doctors a rep visits matters as much as how many.
  • Isolated a 99-doctor "high-potential growth" segment to prioritise; rep output ranged from 6,007 scripts down to 154 — a clear coaching signal. Turned "visit more doctors" into a ranked list of 99 to visit first.
  • Shipped as a notebook plus a live Tableau sales-performance dashboard.
  • Python

  • statsmodels

Battle — turn-based RPG with online PvP Vanilla JS · Supabase

A Pokémon-style RPG with story mode, persistent saves and real-time online multiplayer, on a custom engine built from the ground up — the project I've put the most into.

  • A hand-written vanilla-JS engine: turn loop, move resolution and the status / hazard system all written by hand, no game framework, so I know exactly how every edge case resolves.
  • An authoritative Supabase / PostgreSQL backend with migrations handles the hard part — two players acting on one shared, authoritative state, plus saved progress — where most of the engineering effort lives.
  • A full story mode (catch, PC, Underground and Safari systems, boss arcs); HP and status restore between battles, but consumables only matter inside an encounter, so every fight is its own resource puzzle and rewards stay fair between strangers in PvP.
  • Hardened like production software: performance profiling of the turn loop and memory, a central issue ledger with spec-drift checks against the design docs, and a PWA manifest.
  • The part I'm proudest of — a test suite built on deterministic replay: the same seed reproduces a run exactly, so a bug someone hits once becomes a test I can replay forever.
  • Vanilla JS engine
  • Supabase / PostgreSQL
  • deterministic replay
  • performance profiling
  • PWA
RocketIO — real-time hex-grid strategy Vanilla JS · Canvas · WebAudio

14 factions (70 units), a missile economy and adaptive AI on a procedurally generated map — tiles, units and sound all generated in code, no engine and no art assets.

  • Real-time strategy on an HTML5 Canvas with almost nothing as a fixed asset: no image files, no pre-built maps, no recorded music — the game assembles its world in code at runtime.
  • Procedural maps from gradient noise, tuned into recognisably different worlds (continents, archipelagos, inland seas), so strategy changes with the geography instead of a fixed board.
  • A missile economy (factories build, launchers fire, anti-air intercepts) read through fog of war, with up to six doctrine-driven AI opponents that re-weigh their plans as the state shifts and can't memorise an opening.
  • Tuned from data, not feel: a separate balance-simulator plays factions against each other headless; Vite bundles, Vitest + GitHub Actions run on every push, and the score is generated adaptively through WebAudio.
